SUB-EPIC: Configure Skill Ranks

STORY: Assign Skill Ranks (user)

Domain terms:
- Skill Rank -- numeric investment; bonus on checks
- Power Point (PP) -- 1 PP buys 2 ranks
- Trained Status -- active when rank >= 1
- Skill Modifier Limit -- max total modifier = PL + 10

1. WHEN Player allocates skill ranks
   THEN the system records each rank against the nominated skill
   AND deducts 1 PP per 2 ranks
   AND activates Trained Status on any skill receiving >= 1 rank
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

2. WHEN Player splits 1 PP across two skills
   THEN both skills receive 1 rank
   AND both gain Trained Status
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

3. WHEN allocation would push total modifier above PL + 10
   THEN the system blocks the assignment
   AND shows current modifier and limit
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

4. WHEN Player leaves a skill at 0 ranks
   THEN the skill remains untrained
   AND ability modifier still applies for non-Trained-Only skills
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

5. BUT allocating ranks to one skill does not reduce ranks on another
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

STORY: Enforce Skill Modifier Limit (system)

Domain terms:
- Skill Modifier Limit -- total modifier (rank + ability + misc) <= PL + 10
- Power Level (PL) -- GM-set campaign ceiling
- Circumstance Modifier -- situational; applied at check time not build time

1. WHEN any skill's total modifier reaches PL + 10 + 1
   THEN the system rejects further rank allocation
   AND reports total and limit
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

2. WHEN GM increases campaign PL
   THEN the system recalculates limits for all skills
   AND unlocks previously capped allocations
   Evidence: HeroesHandbook-rules__chunk_024.md lines 1650-1767

3. WHEN a circumstance bonus temporarily pushes check total above limit
   THEN the system applies it for that single check
   BUT does not alter stored rank or modifier
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

4. BUT the limit applies per-skill independently -- not cumulative across all skills
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

SUB-EPIC: Resolve Skill Checks

---

STORY: Make Skill Check (user)

Domain terms:
- Skill Check -- d20 + rank + ability mod + circumstance mods vs DC
- Difficulty Class (DC) -- threshold; equal or above = success
- Degree of Success/Failure -- (total-DC)/5 or (DC-total)/5

1. WHEN Player declares a skill use and DC is established
   THEN system computes total = d20 + rank + ability + circumstances
   AND compares to DC
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735; chunk_042.md lines 2736-2796

2. WHEN total >= DC
   THEN system records success
   AND degree of success = (total-DC)/5 rounded down
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

3. WHEN total < DC
   THEN system records failure
   AND degree of failure = (DC-total)/5 rounded down
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

4. WHEN skill is Trained Only and rank = 0
   THEN attempt automatically fails, no roll made
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

5. BUT one DC applies to the entire check
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

STORY: Apply Critical Success on Natural 20 (system)

Domain terms:
- Natural 20 -- d20 face showing 20
- Critical Success -- degree increased by 1 after normal calculation
- Degree of Success -- graded result

1. WHEN d20 result is exactly 20
   THEN system calculates normal degree first
   AND increases that degree by one
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

2. WHEN natural 20 on already-maximum-degree check
   THEN degree stays at maximum
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

3. WHEN natural 20 on a failing check
   THEN system applies degree increase after failure calculation
   AND may convert failure to success if boost crosses zero
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

4. BUT natural 20 is not automatic success -- degree math runs first
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

STORY: Resolve Untrained Skill Attempt (system)

Domain terms:
- Trained Only -- skill flag; rank 0 = auto-fail
- Untrained -- character has rank 0
- Ability Modifier -- always applies even at rank 0

1. WHEN Player attempts Trained Only skill with rank 0
   THEN system automatically fails, no roll made
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

2. WHEN Player attempts non-Trained-Only skill with rank 0
   THEN system computes total = d20 + 0 + ability + circumstances
   AND resolves normally
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

3. WHEN skill has distinct trained/untrained aspects
   THEN untrained characters may attempt only the untrained aspects
   AND system blocks trained-aspect attempts
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

4. BUT Trained Only is a hard block, not an attempt-at-penalty
   Evidence: HeroesHandbook-rules__chunk_042.md lines 2736-2796

STORY: Apply Skill Mastery Routine Result (system)

Domain terms:
- Skill Mastery -- advantage granting routine result without rolling
- Routine Check Result -- modifier + 10
- Nominated Skill -- specific skill assigned to Skill Mastery

1. WHEN Player has Skill Mastery and attempts the nominated skill under stress
   THEN system returns modifier + 10 without rolling
   Evidence: HeroesHandbook-rules__chunk_043.md lines 2797-2849

2. WHEN Skill Mastery applies
   THEN routine result is final; no roll to improve it
   Evidence: HeroesHandbook-rules__chunk_043.md lines 2797-2849

3. WHEN nominated skill would require a check in combat
   THEN Skill Mastery applies; no roll needed
   Evidence: HeroesHandbook-rules__chunk_059.md lines 3762-3826

4. BUT Skill Mastery does not apply to other skills
   Evidence: HeroesHandbook-rules__chunk_043.md lines 2797-2849

5. BUT routine result still uses capped modifier
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

STORY: Apply Circumstance Modifier to Skill Check (system)

Domain terms:
- Circumstance Modifier -- situational bonus or penalty
- Requires Tools -- skill flag; lacking tools = -5
- Stacking -- multiple circumstance modifiers add together

1. WHEN favorable condition applies
   THEN system adds circumstance bonus to check total
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

2. WHEN required tools are absent for Requires Tools skill
   THEN system applies -5 circumstance penalty
   Evidence: HeroesHandbook-rules__chunk_043.md lines 2797-2849

3. WHEN multiple circumstance modifiers apply
   THEN all stack additively
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735

4. BUT circumstance modifiers do not permanently alter rank or ability modifier
   Evidence: HeroesHandbook-rules__chunk_041.md lines 2689-2735
